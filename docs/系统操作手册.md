## 1. 系统概述

### 1.1 系统简介

鼠小弟的健身系统是一套基于Excel存储的健身房经营管理系统，提供会员管理、员工管理、售课、排课、预约、充值、商品零售、**课后服务（会员分析+流失预警）**、**合同管理**、**财务管理**、报表导出等完整的经营功能。V2.5.0版本新增了**连锁多门店管理**功能，支持多个门店的数据隔离与统一管理。V2.6.0版本新增了**课后服务**、**合同管理**、**财务管理**三大模块。

### 1.2 系统架构

```
┌──────────────────────────────────────────┐
│              GUI 界面层                     │
│  (Tkinter / 20+ 管理界面模块)               │
├──────────────────────────────────────────┤
│             业务逻辑层                      │
│  (BusinessLayer / 数据验证 / 权限管理)      │
├──────────────────────────────────────────┤
│             数据引擎层                      │
│  (ExcelEngine / openpyxl封装)              │
├──────────────────────────────────────────┤
│            Excel 数据文件                   │
│  (17个Sheet / 212个数据字段)               │
└──────────────────────────────────────────┘
```

### 1.3 核心特性

| 特性 | 说明 |

## 2. 快速开始

### 2.1 启动系统

```bash
python main.py
```

或使用提供的启动脚本：

```bash
python scripts/launch.py
```

### 2.2 系统界面布局

启动后将看到：

1. **左侧导航栏** — 所有功能模块入口
2. **顶部门店选择器** — 切换当前管理的门店（多门店场景）
3. **中央工作区** — 当前功能模块的操作界面

### 2.3 数据备份

系统每次启动前自动备份数据文件到 `data/backup/` 目录。如需手动备份，可从菜单操作。

### 2.4 初始化系统

首次使用请运行初始化脚本创建完整的Sheet结构：

```bash
python scripts/init_workbook.py
```

## 4. 连锁门店管理

### 4.1 门店管理

**入口**: 左侧导航 → 连锁门店 → 门店管理

**功能**:
- **新增门店** — 填写门店名称、地址、电话、店长、营业时间等
- **编辑门店** — 双击或右键编辑门店信息
- **删除门店** — 删除门店会同时清除该门店的数据映射
- **门店状态** — 营业中 / 休息中 / 装修中 / 已关闭

门店自动编号格式：`ST001`、`ST002`...

### 4.2 数据映射机制

系统采用**映射表方案**实现门店数据隔离，而非修改原始数据结构：

```
会员数据 ← 门店数据映射表 → 门店1
课程数据 ← 门店数据映射表 → 门店2
售课数据 ← 门店数据映射表 → 门店3
```

**优势**：
- 17个Sheet的原始数据结构不受影响
- 一条数据可属于多个门店（未来扩展）
- 支持门店间数据转移和合并

### 4.3 跨店统计

**入口**: 左侧导航 → 连锁门店 → 跨店统计

显示所有门店的经营数据对比：

| 门店 | 状态 | 会员数 | 本月售课 | 本月充值 | 本月零售 | 上课数 |
|:----|:---:|:-----:|:-------:|:-------:|:-------:|:-----:|
| 总店 | 营业中 | 50 | ¥15,000 | ¥8,000 | ¥3,200 | 120 |

### 4.4 权限管理

**入口**: 左侧导航 → 连锁门店 → 权限管理

**四种角色**：

| 角色 | 图标 | 权限范围 | 适用对象 |
|:----|:---:|:---------|:--------|
| 超级管理员 | 🔴 | 所有门店全部数据读写 | 系统管理员、老板 |
| 门店经理 | 🔵 | 仅管辖门店的数据读写 | 店长 |
| 普通员工 | 🟢 | 仅管辖门店的数据查看 | 前台、教练 |
| 审计员 | 🟡 | 所有门店数据只读 | 财务、总部 |

**用户管理**:
- **新增用户** — 设置姓名、账号、密码、角色和管辖门店
- **编辑/删除用户** — 双击用户或右键操作
- **管辖门店** — 多个门店用逗号分隔（ST001,ST002）

### 4.5 数据转移

**入口**: 左侧导航 → 连锁门店 → 数据转移

**使用场景**：当门店调整、合并或数据归属变更时

**操作流程**：
1. 选择**源门店**（数据当前所在门店）
2. 选择**数据类型**（会员/员工/售课等）
3. 从列表选中要转移的数据（可多选/全选）
4. 选择**目标门店**
5. 点击「执行转移」

**门店合并**：将源门店所有数据转移到目标门店，并关闭源门店。

**转移日志**：所有转移操作自动记录，可在右侧面板查看最近记录。

### 4.6 门店数据导出

**入口**: 左侧导航 → 连锁门店 → 门店导出

**两种导出模式**：

1. **门店数据导出** — 按门店+数据类型筛选，支持日期范围
2. **门店汇总报表** — 一键导出所有门店经营汇总对比

支持的数据类型：会员、员工、售课、上课、课程包、预约、商品、零售、体测、充值、日志

## 5. 基础数据管理

### 5.1 会员管理

**入口**: 左侧导航 → 基础数据 → 会员管理

**支持的字段**: 会员编号、姓名、性别、手机号、生日、身份证、加入日期、会员类型、课程包、剩余课时、总充值、备注等

**功能**:
- **新增会员** — 填写会员信息表单
- **编辑会员** — 双击会员行进行修改
- **删除会员** — 支持一键删除
- **搜索筛选** — 按姓名/手机号/会员类型搜索
- **分页显示** — 支持大数据量分页查看

### 5.2 员工管理

**入口**: 左侧导航 → 基础数据 → 员工管理

支持员工信息管理，包括岗位（教练、前台、销售、店长、保洁等）、入职日期、薪资、电话等。支持搜索和筛选。

### 5.3 课程管理

**入口**: 左侧导航 → 基础数据 → 课程管理

管理课程项目：课程编号、名称、类型（私教/团课/小班课/体验课/康复课）、价格、课时数、教练等。

### 5.4 商品管理

**入口**: 左侧导航 → 基础数据 → 商品管理

支持商品信息管理：商品编号、名称、类别（蛋白粉/增肌粉/氮泵/护具等10种）、进价、售价、库存、单位。

## 6. 课后服务

### 6.1 会员分析

**入口**: 左侧导航 → 课后服务 → 会员分析

会员分析提供会员维度的深度画像，帮助教练和管理人员了解每位会员的运动习惯和健康趋势。

**核心功能**:

1. **会员选择** — 从下拉列表选择一个会员，自动加载该会员的完整画像
2. **会员概览** — 展示会员等级、剩余课时、总消费金额等关键指标
3. **上课趋势（折线图）** — 展示近7天/30天/90天的上课频率变化趋势
4. **课程偏好** — 按课程类型统计上课次数占比（私教/团课/其他）
5. **体测趋势** — 展示体重和体脂率的历史变化曲线，直观反映训练效果
6. **流失风险评估** — 根据连续未上课天数、剩余课时、最近到店时间综合评估流失风险等级

### 6.2 流失预警

**入口**: 左侧导航 → 课后服务 → 流失预警

自动识别有流失风险的会员，帮助主动跟进和挽留。

**风险等级规则**:

| 风险等级 | 颜色 | 判断条件 | 建议措施 |
|:--------|:----:|:---------|:---------|
| 🟢 低风险 | 绿色 | 连续未上课 14-30 天 | 发送提醒消息、推送优惠 |
| 🟡 中风险 | 橙色 | 连续未上课 30-60 天 | 电话回访、安排私教体验课 |
| 🔴 高风险 | 红色 | 连续未上课 > 60 天 | 店长亲自跟进、赠送课程包 |

**附加标记**:
- 剩余课时 ≤ 3 时，表格行会**高亮显示**，提示课时不足
- 每条记录显示**原因分析**

**操作**:
- 点击「查看画像」跳转到该会员的详细分析页
- 可一键发送回访提醒

## 7. 业务记录

### 7.1 售课记录

**入口**: 左侧导航 → 业务记录 → 售课记录

记录课程销售信息：会员、课程、金额、支付方式、经办人、备注等。支持按日期和会员搜索。

### 6.2 课程包管理

**入口**: 左侧导航 → 业务记录 → 课程包

管理会员购买的课程包：关联会员、课程、总课时、已消耗课时、有效期等。自动计算剩余课时和到期提醒。

### 6.3 预约管理

**入口**: 左侧导航 → 业务记录 → 预约管理

完整的预约排课系统：

- **日期导航** — 左右翻页、一键回到今天
- **按日期筛选** — 查看指定日期的预约列表
- **新增预约** — 选择会员、教练、时段
- **冲突检测** — 同一教练/会员在同一时段只能有一个"已预约"记录
- **签到** — 到店签到自动消耗课时、创建上课记录
- **完成课程** — 标记预约状态为已完成
- **取消预约** — 取消预约不扣课时
- **批量签到** — 团课/小班课多人批量签到

**预约状态机**:
```
已预约 → 已签到 → 已完成
   ↓
已取消
```

### 6.4 上课记录

**入口**: 左侧导航 → 业务记录 → 上课记录

记录每次上课信息：上课编号、会员、教练、课程、日期、时长、备注。可由预约签到自动创建。

### 6.5 会员充值

**入口**: 左侧导航 → 业务记录 → 会员充值

记录会员充值记录：充值编号、会员、充值金额、充值方式、赠送金额、经办人。

### 6.6 商品零售

**入口**: 左侧导航 → 业务记录 → 商品零售

商品售卖记录：零售编号、商品名称、数量、单价、总价、会员、经办人。**库存自动扣减**。

## 7. 合同管理

**入口**: 左侧导航 → 合同管理

合同管理提供完整的合同生命周期管理，从创建到签署到作废的全流程支持。

### 7.1 合同列表

主界面以表格展示所有合同，包含以下信息：

| 字段 | 说明 |
|:----|:------|
| 合同编号 | 自动生成（CT+年月日+4位序号） |
| 合同类型 | 入会协议 / 私教合同 / 商品购买 / 免责声明 / 其他 |
| 会员姓名 | 合同关联的会员 |
| 合同金额 | 合同约定的金额（¥） |
| 签署状态 | 待签署 / 已签署 / 已作废 |
| 签署日期 | 实际签署日期 |
| 有效期起止 | 合同的有效时间范围 |
| 签署方式 | 纸质 / 电子 / 扫码 |

### 7.2 新增合同

点击「➕ 新增合同」按钮，弹出合同编辑对话框：

- **合同类型** — 从预设类型中选择
- **会员信息** — 填写会员编号和会员姓名
- **合同金额** — 填写合同金额（数字）
- **有效期** — 合同生效日期和截止日期
- **签署方式** — 纸质/电子/扫码

### 7.3 签署与作废

- **签署** — 选中一条「待签署」合同，点击「✅ 签署」确认签署
- **作废** — 选中合同，点击「❌ 作废」可将合同标记为作废状态
- 已签署的合同不可重复签署

### 7.4 搜索

顶部搜索框支持按**合同编号、会员姓名、合同类型**实时搜索，快速定位合同。

### 7.5 到期提醒

页面底部自动显示**近30天内到期**的合同：
- 🔴 **已到期** — 合同已过有效期
- 🟡 **即将到期** — 30天内即将到期的合同

### 7.6 合同统计

页面右上角显示合同统计概览：合同总数、已签署数量、待签署数量、已作废数量。

## 8. 财务管理

财务管理模块提供收入汇总、支出管理和经营报表三大功能，支持日报/月报/年报的多维度财务分析。

### 8.1 收入总账

**入口**: 左侧导航 → 财务管理 → 收入总账

自动汇总所有收入来源，无需手动录入大部分数据。

**自动同步**:

收入总账自动从以下三个来源汇总数据（点击「🔄 同步数据」触发）：

| 来源 | 类型 | 数据源 |
|:----|:----:|:-------|
| 售课记录 | 售课收入 | 已售课程的实际收款 |
| 会员充值 | 充值收入 | 会员充值实付金额 |
| 商品零售 | 零售收入 | 商品零售总价 |

**操作**:
- **年月筛选** — 选择年份和月份后点击「筛选」
- **一键同步** — 从各业务Sheet自动拉取当月收入记录
- **统计信息** — 右上角显示当月收入总额和记录数

### 8.2 支出管理

**入口**: 左侧导航 → 财务管理 → 支出管理

记录和管理健身房的所有支出。

**支持支出类别**:

| 类别 | 说明 | 示例 |
|:----|:-----|:-----|
| 房租 | 门店租金 | 月租、物管费 |
| 水电 | 水电费用 | 电费、水费、网络费 |
| 工资 | 员工薪资 | 基本工资、绩效 |
| 设备 | 设备采购 | 健身器材、维修 |
| 装修 | 装修费用 | 翻新、改造 |
| 营销 | 营销推广 | 广告、活动 |
| 办公 | 办公用品 | 文具、耗材 |
| 其他 | 其他支出 | 自定义 |

**操作**:
- **新增支出** — 填入日期、类别、金额、支付方式、经办人
- **年月筛选** — 选择年份和月份查看
- **统计信息** — 右上角显示当月支出总额和记录数

### 8.3 财务报表

**入口**: 左侧导航 → 财务管理 → 财务报表

提供经营数据的总览和深度分析。

**模式切换**:
- **月报** — 查看指定月份的收入和支出明细
- **年报** — 查看全年12个月的月度趋势

**摘要卡片**（顶部的4张卡片）：

| 卡片 | 说明 |
|:----|:------|
| 总收入 | 当月/年所有收入之和 |
| 总支出 | 当月/年所有支出之和 |
| 利润 | 总收入 - 总支出（绿色表示盈利，红色表示亏损） |
| 利润率 | 利润/总收入的百分比，直观反映盈利水平 |

**详细报表**（底部Tab面板）：

| Tab | 内容 |
|:----|:------|
| 收入明细 | 按收入类型（售课/充值/零售）分类汇总金额 |
| 支出明细 | 按支出类别（房租/水电/工资等）分类汇总金额 |
| 月度趋势 | 年报模式下显示全年12个月的收入、支出、利润月度趋势表 |

## 12. 常见问题

### Q1: 数据文件在哪？

数据文件位于 `data/健身房会员系统.xlsx`，所有数据存储其中。建议定期备份。

### Q2: 如何备份数据？

系统在 `data/backup/` 目录自动创建备份。也可以手动复制Excel文件到安全位置。

### Q3: 如何切换到另一个门店？

在界面左侧导航栏上方选择门店下拉框切换。切换后所有模块自动显示对应门店数据。

### Q4: 新增门店后已有数据如何关联？

门店管理器中会自动创建**默认门店**，并将所有现有数据关联到该门店。新数据由用户选择门店。

### Q5: 支持多少门店？

理论无上限，受限于Excel文件大小。实际建议不超过50个门店。

### Q6: 数据从旧系统迁移？

使用 **「门店数据转移」** 功能可以灵活分配数据归属。如需整体数据导入，请联系开发者。

### Q7: 密码忘记了怎么办？

在权限管理中，管理员可以直接编辑用户密码。首次使用默认密码 123456。

### Q8: 如何对接微信小程序？

启动API服务器后，小程序端通过HTTP请求调用 `/api/*` 接口。支付接口已预留，后续对接支付网关即可。
