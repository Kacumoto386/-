# gym-excel-system 系统优化 Phase2 方案

## 背景
- **版本**: V2.8.5
- **Phase1**（代码重构）已完成：business.py 拆分为 10 个 mixin 模块
- **当前代码状态**: 所有源文件编译通过，23/24 模块导入通过

## Phase2 优化目标：错误处理增强 + UI/UX 改进

### 1. 全局异常处理统一化

**问题**: 目前创建/编辑弹窗中的异常处理分散，有的被 `try/except` 静默吞掉，导致白屏或按钮无响应。

**方案**:
1.1 创建 `core/exceptions.py` — 自定义异常层次
1.2 在 `gui/base_frame.py` 的 CRUD 方法中统一异常捕获
1.3 所有弹窗对话框的 `on_save()` 统一异常处理

### 2. 用户操作反馈增强

**问题**: 部分操作（删除、批量操作）无确认对话框，无成功提示。

**方案**:
2.1 删除操作统一确认对话框（`messagebox.askyesno`）
2.2 成功/失败操作统一 toast 提示
2.3 数据刷新后显示更新状态

### 3. UI 加载状态与搜索增强

**问题**: 大型数据集加载时无进度反馈，搜索功能不统一。

**方案**:
3.1 长时间加载显示 "加载中..." 提示
3.2 统一搜索栏组件（`SearchBar` 类）
3.3 所有表格页增加搜索功能

### 4. 数据验证统一化

**问题**: 验证逻辑分布在各个模块的 `on_save()` 中，风格不一致。

**方案**:
4.1 增强 `core/validator.py` — 增加通用验证函数
4.2 统一的表单验证入口

### 5. 代码清理

**问题**: 存在废弃的商店管理代码和冗余 import

**方案**:
5.1 删除 `__pycache__` 目录
5.2 清理 `stability_test.py` 分散在各目录的副本
5.3 检查并移除引用 store_mgr 的废弃代码

## 执行顺序

1. 创建 `core/exceptions.py`
2. 增强 `gui/base_frame.py` 异常处理
3. 所有弹窗统一确认/提示
4. 搜索栏组件
5. 数据验证增强
6. 代码清理

## 测试策略
- 每项修改后运行全模块编译检查
- Phase2 完成后运行稳定性测试
