## 一、课后服务模块（V2.6.0a）

### 痛点
- 目前只有流水记录（售课、上课），缺少**会员维度的深度分析**
- 教练只能看售课排名，不知道哪个会员要流失了
- 体测数据孤立，没有趋势可视化

### 设计方案

#### 1. 会员画像看板（新页面）

**数据来源**: 现有 member / class_record / sale / body_measurement

**展示内容**:

```
┌──────────────────────────────────────┐
│ 会员画像 · 张三                        │
│ 会员等级: 金卡 | 剩余课时: 25 | 总消费: ¥15,000 │
├─────────┬────────────────────────────┤
│ 上课趋势  │ 近7天 | 近30天 | 近90天     │
│ (折线图) │ ▁▃▅▆▄▅▇                    │
├─────────┼────────────────────────────┤
│ 课程偏好  │ 私教: 60% | 团课: 30% | 其他: 10% │
│ (饼图)   │                            │
├─────────┼────────────────────────────┤
│ 体测趋势  │ 体重▁▁▁▂▃ 体脂▇▅▃▂▁        │
│ (折线图) │ 趋势: 向好 ↑               │
├─────────┼────────────────────────────┤
│ 流失风险  │ 风险等级: ⚠️ 中             │
│          │ 连续未上课: 12天            │
│          │ 剩余课时: 25 (够用)         │
│          │ 上次到店: 2026-04-22        │
└─────────┴────────────────────────────┘
```

**新增文件**:
- `core/member_analysis.py` — 会员分析引擎

| 方法 | 说明 |
|:----|:------|
| `get_member_profile(member_id)` | 会员画像（上课趋势/课程偏好/体测趋势） |
| `get_churn_risk(member_id)` | 流失风险评估（连续缺席天数、剩余课时、最近签到） |
| `get_training_trend(member_id, days)` | 上课频率趋势数据 |

#### 2. 流失预警看板（新页面）

**规则**:
- 连续未上课 > 14天 → ⚠️ 低风险
- 连续未上课 > 30天 → 🚨 中风险
- 连续未上课 > 60天 → 🔴 高风险
- 剩余课时 ≤ 3 → 叠加课时不足标记

**新增 GUI**:
- `gui/member_analysis_frame.py` — 会员分析/画像UI
- `gui/churn_warning_frame.py` — 流失预警看板（列表+风险标签）

**Sidebar 入口**:
```
课后服务 →
  ├── 会员分析（画像数据）
  ├── 流失预警（风险列表）
  └── [保留]
```

#### 3. 需要改动的文件

## 六、设计原则

1. **不破坏现有数据** — 所有新增Sheet不修改已有17个Sheet结构
2. **自动同步优先** — 财务收入从已有数据自动汇总，减少手动录入
3. **模块化** — 三个模块互相独立，可以按任意顺序开发
4. **可扩展** — 财务模块预留了「其他收入」「其他支出」自定义分类
