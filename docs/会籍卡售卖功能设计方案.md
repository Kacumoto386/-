# 会籍卡售卖功能设计方案

## 一、需求概述

在**会员管理模块**中，录入会员基础信息后，可以直接操作为该会员售卖「会籍卡」。会籍卡是会员权益的载体，替代/补充现有的"课程包"体系。

## 二、会籍卡类型设计

### 三种卡类型

| 类型 | 核心逻辑 | 业务样例 |
|------|---------|---------|
| **次卡** `type=count` | 购买N次，每次签到扣1次，用完为止 | 30次健身卡、20次私教卡 |
| **期限卡** `type=period` | 固定期限内无限次使用，到期自动失效 | 月卡、季卡、年卡、晨练卡、暑期卡 |
| **现金卡** `type=cash` | 充值金额，消费时扣款，类似储值卡 | 储值1000元卡、充值500元卡 |

### 三种卡的共同字段

```
会籍卡编号, 会员编号, 会员姓名, 卡类型(次/期/现), 卡名称,
售价, 实收金额, 付款方式, 销售员工, 开卡日期,
状态(有效/已用完/已过期/已退费), 操作员, 备注, 门店编号
```

### 三种卡的差异化字段

| 字段 | 次卡 | 期限卡 | 现金卡 |
|------|:----:|:------:|:------:|
| 总次数 | ✅ | - | - |
| 已消耗次数 | ✅ | - | - |
| 剩余次数 | ✅ | - | - |
| 有效期起 | ✅ | ✅ | - |
| 有效期止 | ✅ | ✅ | - |
| 有效天数 | - | ✅ | - |
| 余额 | - | - | ✅ |
| 已消费金额 | - | - | ✅ |

**设计决策**：所有字段合并到一个Sheet中，未用字段留空。这样便于统计和查询，结构统一。

## 三、数据流设计

### 3.1 创建会籍卡后的联动

| 卡类型 | 购买时 | 使用时 |
|--------|-------|--------|
| **次卡** | 会员剩余课时 += 总次数 | 签到/上课时：剩余次数 - 1，会员总消耗 + 1 |
| **期限卡** | 会员状态 = 有效，有效期同步更新 | 系统自动检测过期，过期后标记"已过期" |
| **现金卡** | - | 消费时扣余额 |

### 3.2 财务流向

```
售卡 → 会籍卡售卖记录 ← 自动 → 收入总账（会籍卡收入项）
               ↓
      次卡：消耗时 → 不影响财务（消耗不是收入事件）
      期限卡：到期 → 不影响财务
      现金卡：消费 → 记录消费明细
```

### 3.3 报表影响

- **首页看板**：增加"本月会籍卡销售额"指标
- **财务报表**：收入总账增加"会籍卡收入"自动汇总项
- **员工提成**：会籍卡销售可以纳入销售提成统计

## 四、用户交互流程

### 4.1 入口设计

**方式A（推荐）**：会员管理页 → 选中某会员 → 点击「🃏 售卡」按钮 → 弹出会籍卡售卖对话框

```
会员管理列表
├── [➕ 新增] [✏️ 编辑] [🗑️ 删除] [🔄 刷新] [🃏 售卡] [📋 会籍卡]
```

即在 `MemberFrame` 上增加两个按钮：
- **「🃏 售卡」** — 为选中会员开售会籍卡
- **「📋 会籍卡」** — 查看选中会员的所有会籍卡

**方式B**：在业务记录区增加独立菜单入口「🃏 会籍卡售卖」，类似"售课记录"

### 4.2 售卡对话框流程

```
1. 自动带入选中会员信息（会员编号、姓名、手机号）
2. 选择卡类型：[次卡 / 期限卡 / 现金卡]  → 动态显示对应字段
3. 选择卡名称（如"30次健身卡"、"年卡"等）→ 自动填入建议售价
4. 填写/确认金额、付款方式、销售员工等
5. 确认开卡 → 生成会籍卡记录 + 更新会员信息（次卡加课时）
```

### 4.3 会员详情页查看会籍卡

```
会员名下的所有会籍卡列表：
┌──────┬──────┬──────┬──────┬──────┬──────┬──────┬──────┐
│ 编号 │ 类型 │ 名称 │ 状态 │ 次数 │ 余额 │ 开卡 │ 操作 │
├──────┼──────┼──────┼──────┼──────┼──────┼──────┼──────┤
│ MC01 │ 次卡 │30次卡│有效  │ 18/30│  -   │5/7  │退费  │
│ MC02 │期限卡│年卡  │有效  │   -  │  -   │5/7  │续费  │
│ MC03 │现金卡│500   │有效  │   -  │320   │5/7  │退费  │
└──────┴──────┴──────┴──────┴──────┴──────┴──────┴──────┘
```

## 七、数据联动逻辑

### 7.1 售卡时对会员的影响

| 卡类型 | 对会员信息的影响 |
|--------|----------------|
| 次卡 | 会员「剩余课时」+= 总次数、「总购课时」+= 总次数 |
| 期限卡 | 会员「会员等级」= 卡名称、「会员状态」= 有效、「到期日期」= 有效期止 |
| 现金卡 | 无直接影响（会员的「剩余金额」可以考虑累加） |

### 7.2 消耗/使用逻辑

**次卡消耗（上课签到）**：
```
签到上课时 → 检查会员名下 有效的次卡 → 优先消耗最早开的次卡
→ 该次卡「已消耗次数」+1、「剩余次数」-1
→ 如果剩余次数=0 → 状态改为「已用完」
→ 会员「已消耗课时」+1、「剩余课时」-1
```

**期限卡过期检查（自动任务）**：
```
每日检查 → 有效期止 < 今天 → 状态改为「已过期」
→ 对应会员的会员状态按原有逻辑处理
```

**现金卡消费**：
```
商品零售时 → 可选用现金卡支付 → 卡余额 -= 消费金额
→ 已消费金额 += 消费金额
→ 余额=0 → 状态改为「已用完」
```

### 7.3 退费逻辑

```
退费 → 卡状态改为「已退费」→ 回滚对会员的影响：
  次卡：会员剩余课时 -= 剩余次数
  期限卡：会员等级/到期日期回退
  现金卡：如已消费，需结算
```

## 八、与现有系统的关系

| 现有概念 | 新概念「会籍卡」 | 关系 |
|----------|----------------|------|
| 课程包 | 次卡 | 相似但更通用。课程包绑定课程，次卡不绑定。**初期可并行存在，未来可考虑合并** |
| 售课记录 | 会籍卡售卖 | 不同业务线。售课记录 = 卖课程，会籍卡 = 卖会员资格 |
| 会员充值 | 现金卡 | 充值只是储值，现金卡更灵活（可退费、可消耗） |
| 会员等级/有效期 | 期限卡 | 期限卡直接对应会员的等级和有效期 |
