# V2.15.1 业绩统计模块设计方案

## 一、现状分析

当前"统计分析"模块包含 3 个子页面：
1. **售课统计** — 售课金额/节数/订单数等卡片指标 + 明细表格
2. **上课统计** — 上课节数/消耗课时/学员数等 + 课程频次表
3. **员工提成** — 按员工统计售课提成/上课提成

现有问题：
- ❌ 只有售课数据，没有**课程包**业绩统计
- ❌ 没有**会籍卡**业绩统计（售卡金额/开卡数等）
- ❌ 没有**会员进场记录**统计（客流量/高峰期等）
- ❌ 模块名称"统计分析"不够直观，建议改为"**业绩统计**"

## 二、改造方案

### 2.1 模块改名
`统计分析` → `业绩统计`（导航栏文字 + 类名 + 文件名）

### 2.2 新增 3 个统计页面

| # | 页面 | 数据源 | 核心指标 |
|:-:|------|--------|----------|
| 1 | **课程包统计** | `课程包管理` Sheet | 总售卖数量、总课时数、已消耗课时、剩余课时、过期数量 |
| 2 | **会籍卡统计** | `会籍卡管理` Sheet + `会员信息` | 售卡总数、总售卡金额、按卡类型统计、新开卡趋势 |
| 3 | **进场统计** | `进场记录` Sheet | 总进场人次、日均客流、高峰时段、核销方式分布 |

### 2.3 现有页面增加

**售课统计**增加：
- 到期时间数据展示（即将过期/已过期的售课数量统计）
- 按课程包状态统计（有效/已用完/已过期的分布）

### 2.4 导航栏结构调整

```
─── 业绩统计 ───
📊  售课统计        ← 增强（含到期数据）
📦  课程包统计      ← 新增
🃏  会籍卡统计      ← 新增
📈  上课统计        ← 现有
🏪  进场统计        ← 新增
💵  员工提成        ← 现有
```

## 三、详细设计

### 3.1 课程包统计（PackageStatsFrame）

**数据源**：`课程包管理` Sheet（SHEETS["lesson_package"]）

**卡片指标**（6个）：
- 总课程包数
- 总售卖课时数（所有课程包的总课时之和）
- 总已消耗课时
- 总剩余课时
- 消耗率（已消耗/总课时）
- 过期课程包数

**课程包表格**：按课程名称/会员 分组的统计

### 3.2 会籍卡统计（MembershipStatsFrame）

**数据源**：`会籍卡管理` Sheet（SHEETS["membership"]）

**卡片指标**（6个）：
- 总售卡数
- 总售卡金额
- 本月售卡数
- 本月售卡金额
- 有效卡数
- 过期卡数

**按卡类型分布**：次卡/期限卡/现金卡的售卡金额占比

**表格**：售卡明细列表（会籍卡编号/会员/卡类型/售价/有效期/状态）

### 3.3 进场统计（CheckinStatsFrame）

**数据源**：`进场记录` Sheet（SHEETS["checkin"]）

**卡片指标**（6个）：
- 总进场人次
- 本月进场人次
- 日均进场人次（本月）
- 次卡核销次数
- 现金卡扣费次数
- 无卡进场次数

**统计表格**：按日期统计的每日进场趋势 + 核销方式分布
