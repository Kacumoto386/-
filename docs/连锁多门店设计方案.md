# 连锁/多门店管理设计方案 v2.5.0

## 设计原则
1. **最小侵入**：现有数据表尽量不改结构，新增「门店」字段作为附加标识
2. **增量兼容**：单门店用户不受影响，多门店用户只需配置门店信息
3. **Excel友好**：新增专属Sheet，不破坏现有数据格式
4. **UI联动**：顶部全局门店选择器，所有模块自动按门店筛选

## 核心改动

### 1. 新增Sheet
| Sheet名称 | 说明 | 关键列 |
|:---------|:----|:------|
| **门店管理** | 门店基础信息 | 门店编号、名称、地址、电话、店长、状态 |
| **门店统计** | 各门店汇总数据 | 门店名称、会员数、售课额、充值额、零售额 |

### 2. 现有数据表增加「所属门店」字段
最小侵入方案：**不修改HEADERS**，而是在数据写入时自动追加门店信息。
各模块写入数据时从全局门店选择器获取当前门店，自动填充到数据行。

实际做法：在每个业务操作中记录门店信息到操作日志或新增「门店」列。

**更优方案**：使用独立的「门店数据映射」Sheet，记录每条数据所属门店。

```
门店数据映射表
├── 数据类型 (member/staff/sale/…)
├── 数据编号 (M001/S002/…)
├── 门店编号 (ST001)
└── 关联时间
```

这种方式完全不影响现有数据结构，通过映射表实现多门店隔离。

### 3. UI改动
- **顶部门店选择器**：下拉菜单 + 管理按钮
- **各模块自动筛选**：根据选择器显示对应门店数据
- **跨店统计看板**：门店对比报表

### 4. 后端改动
- `BusinessLayer` 新增门店管理方法
- 新增 `core/store_manager.py` 门店管理器
- 所有数据查询增加门店过滤

## 数据映射方案详解

### 为什么选映射表而不是改字段？
1. 现有17个Sheet每个HEADERS都有几十列，改结构风险大
2. 映射表支持一条数据属于多个门店（未来扩展）
3. 映射表可以记录门店变更历史
4. 现有API接口完全不受影响

### 查询流程
```
用户选择门店 → 查询门店映射表获取该门店的数据编号列表
  → 在各Sheet中过滤出这些编号的数据
  → 显示
```

### 写入流程
```
用户选择门店 → 写入数据到原Sheet → 同时在门店映射表添加记录
```

## 跨店统计
- 各门店会员数对比
- 各门店营收对比（售课/充值/零售）
- 各门店教练业绩对比
- 门店月度趋势
