# 功能1：课程包/套餐管理 设计方案

## 需求分析
售课记录没有独立的课程包概念，会员的"总购课时"是累加所有售课记录的课时，无法区分不同课程包。
每次售课后，需要生成一个独立的课程包记录，上课时从对应的课程包扣减课时。

## 设计思路

### 数据层设计

**新增Sheet**：`课程包管理`
- 独立存储每个课程包记录
- 每次售课时自动生成一个课程包

**课程包表头**：
```
课程包编号(PK) | 售课编号 | 会员编号 | 会员姓名 | 课程编号 | 课程名称 | 总课时 | 已消耗课时 | 剩余课时 | 有效期起 | 有效期止 | 状态(有效/已用完/已过期)
```

### 业务层设计（core/business.py）

新增方法：
- `add_lesson_package(data)` - 创建课程包（售课时自动调用）
- `get_member_packages(member_id)` - 获取会员的所有课程包
- `get_package(package_id)` - 获取单个课程包详情
- `get_package_by_sale(sale_id)` - 按售课编号查找课程包
- `consume_lesson(member_id, course_id, qty=1)` - 上课消耗课时，优先从匹配课程包扣减
- `update_package_status()` - 更新课程包状态（检查有效期）
- `_update_member_lessons_with_packages(member_id)` - 基于课程包更新会员课时

### 修改现有逻辑

1. **售课时自动创建课程包**：在 `add_sale()` 中增加调用
2. **上课消耗课时**：在 `add_class_record()` 中优化消耗逻辑
3. **会员课时计算**：`_update_member_lessons()` 改为从课程包汇总

### GUI设计

**新增界面**：`gui/package_frame.py`
- 放在导航栏"业务记录"分组下
- 功能：查看所有课程包、按会员筛选、查看会员维度课程包列表
- 显示每个课程包的消耗进度

## 数据流

```
售课 → add_sale()
  → 创建售课记录
  → add_lesson_package()（自动创建课程包）
  → 更新会员课时

上课 → add_class_record()
  → 找到匹配的课程包
  → 扣减课程包剩余课时
  → 更新上课记录
  → 更新会员课时（从课程包汇总）
```

## 课程包编号规则
PK + 年月日 + 4位序号（参考售课编号格式）
