# 功能3：会员体测记录追踪 设计方案

## 需求分析
会员表有身高/体重/体脂率字段，但没有历史记录。需要支持多次体测记录追踪和曲线展示。

## 设计思路

### 数据层设计

**新增Sheet**：`体测记录`
- 独立存储每次体测数据
- 支持多次记录

**体测记录表头**：
```
体测编号 | 会员编号 | 会员姓名 | 体测日期 | 身高(cm) | 体重(kg) | 体脂率(%) | BMI | 肌肉量(kg) | 基础代谢(kcal) | 体年龄 | 备注
```

BMI 自动计算：体重(kg) / (身高(m))^2

### 业务层设计（core/business.py）

新增方法：
- `add_body_measurement(data)` - 添加体测记录
- `update_body_measurement(row, data)` - 更新体测记录
- `delete_body_measurement(row)` - 删除体测记录
- `get_member_measurements(member_id)` - 获取会员的所有体测记录（按日期排序）
- `get_all_measurements()` - 获取所有体测记录

### GUI设计

**新增界面**：`gui/body_measurement_frame.py`
- 放在"辅助工具"分组下
- 功能：
  - 表格展示所有体测记录
  - 按会员筛选
  - 新增/编辑/删除体测记录
  - 查看会员体测历史曲线

**会员详情中增加体测记录入口**：
- 在会员管理界面中增加"体测记录"按钮
- 点击后跳转到该会员的体测历史

### 体测编号规则
BM + 年月日 + 4位序号
