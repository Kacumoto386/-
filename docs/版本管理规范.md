# 版本管理规范

## 版本号规则

```
v{major}.{minor}.{patch}
```

| 位 | 含义 | 示例 |
|----|------|------|
| major | 大版本重写/架构变更 | v3.0.0 |
| minor | 新增功能模块 | v2.6.0 |
| patch | Bug修复/小改进 | v2.6.1 |

## 每次修复的标准流程

### Step 1: 备份当前版本
```bash
python scripts/backup.py "修复内容简要说明"
```
备份会自动保存到 `_backups/V{版本}_{备注}_{时间戳}/`

### Step 2: 修改版本号
编辑 `config.py`：
```python
__version__ = "2.6.1"  # 递增 patch 号
```

### Step 3: 进行修复
正常修改代码。

### Step 4: 提交记录
确保修复记录更新到 `MEMORY.md`。

## 备份存储结构

```
_backups/
├── V2.6.0_完整存档_20260504_122648/    # 完整版本存档
├── V2.6.1_修复白屏和按钮_20260505_0945/  # 每次修复后备份
└── V2.6.1_门店切换修复_20260505_1147/
```

每个备份文件夹包含：
- 所有源代码文件（core/, gui/, api/ 等）
- 配置文件（config.py）
- 数据文件（data/）
- 文档（docs/）
- 备份信息（_backup_info.json）：版本号、时间、备注

## 回退方法

### 场景：需要回退到某个版本
1. 找到目标备份文件夹：`_backups/V{版本}_{备注}_{时间戳}/`
2. 替换项目文件：
```bash
copy _backups/V2.6.0_完整存档_20260504_122648/config.py projects/gym-excel-system/config.py
copy _backups/V2.6.0_完整存档_20260504_122648/core projects/gym-excel-system/core/
# ... 按需恢复
```

### 场景：完全回退到上一个版本
```bash
python scripts/restore.py  # 后续可实现的恢复工具
```
